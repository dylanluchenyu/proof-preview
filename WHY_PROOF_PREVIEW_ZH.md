# 为什么做 Proof Preview

## 这是什么

Proof Preview 是一个本地、离线的 LaTeX 数学证明预览工具。

它的目标不是替代完整的 LaTeX 排版流程，而是提供一个更轻量、更快的方式，让我在写证明、改证明、看证明的时候，马上看到数学公式和证明结构的实际显示效果。

这个项目目前的核心形式是一个本地 HTML 查看器，加上一个 `proof-preview` 命令行工具。命令会把最新的证明内容写到当前槽位里，浏览器页面会自动刷新显示。

## 为什么我需要它

### 1. LaTeX 在 CLI 或聊天界面里显示得不够好

很多时候，证明内容是在终端、CLI 工具或者聊天界面里生成的。即使模型输出了 LaTeX，阅读体验通常还是不好：

- 行内公式和块公式不容易快速扫读
- 长证明在纯文本界面里层次不清楚
- 推导步骤和公式对齐效果差
- 一旦内容变长，阅读和检查会变得很累

也就是说，模型能写出 LaTeX，不代表这个 LaTeX 在当前界面里“好看”或者“好读”。

Proof Preview 解决的是“显示”这一步：把原本在 CLI 里不容易读的证明，快速变成一个本地网页视图，让公式、标题、段落和结构都更清楚。

### 2. 不想每次都走完整的 `.tex -> 编译 -> PDF` 流程

如果只是想快速检查一个证明写得顺不顺、公式是否易读，其实没必要每次都：

- 新建一个 `.tex` 文件
- 写导言区
- 补各种 package
- 跑 LaTeX 编译
- 再打开 PDF

这套流程对正式文档当然合理，但对“快速看一下刚写出来的证明”来说太重了。

Proof Preview 的定位就是一个更轻的中间层：不用先搭完整工程，也不用先生成正式 PDF，就能先把证明内容看清楚。

### 3. 每次让模型生成完整 LaTeX 文件，会浪费更多 token

如果每次都要求模型输出完整 `.tex` 文件，通常会多出很多和证明本身无关的内容，例如：

- 文档类
- 导言区
- 宏包引用
- 页面设置
- 模板代码

这些内容一方面会消耗更多 token，另一方面也会让真正重要的“证明本身”被模板噪声包住。

Proof Preview 的思路更直接：

- 让模型只输出证明正文
- 直接送进预览器
- 立即查看公式和结构效果

这样更省 token，也更适合反复迭代、来回修改。

### 4. 不想额外安装一整套 LaTeX 环境

很多快速预览场景其实不需要完整 TeX 发行版。安装和维护整套 LaTeX 环境，对“只是想看一下证明显示效果”来说成本偏高。

Proof Preview 走的是另一条路：

- 本地网页渲染
- 仓库里已经打包好需要的 KaTeX 相关资源
- 不依赖在线服务

所以它更像一个“开箱即用的数学显示层”，而不是一个正式排版系统。

## 这个工具最适合什么场景

它特别适合下面这些场景：

- 我在 CLI 里让模型生成数学证明
- 我想快速检查公式显示效果
- 我想低成本反复改写证明措辞和结构
- 我不想每次都生成完整 `.tex`
- 我不想为了预览去启动完整 LaTeX 编译流程
- 我希望内容保留在本地，不依赖在线网站
